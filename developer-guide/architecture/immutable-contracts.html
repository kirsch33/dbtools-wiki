Immutable Production Contracts

Audience: AI agents and contributors. These contracts MUST NOT be changed without explicit user approval.

What Is This Page?

This page lists production behaviors that are performance-critical, safety-critical, or semantically fundamental. Changing them to fix a test failure is almost always wrong — the test is more likely outdated than the contract.

Contracts

1. Usage Index Sampling Cap (GM)

Location: src/Tools/Common/GM/Features/Usage/UsageIndexService.cs — QuerySampleInstanceIdsBySymbol Contract: Maximum 2500 sampled instances. Model-owned first, then view-owned, with early exit. Why: Unbounded iteration causes O(views * instances) scaling. A project with 200 views and 50,000 instances will stall for minutes.

2. Single Geometry Pass per Instance (GM)

Location: src/Tools/Common/GM/Features/Usage/UsageIndexService.cs — QueryStyleUsageChunk Contract: One get_Geometry() call per instance with IncludeNonVisibleObjects = true. Why: get_Geometry() is the most expensive Revit API call. A second pass doubles the cost with no correctness benefit — IncludeNonVisibleObjects already captures all geometry.

3. Single Instance for Parameter Discovery (GM)

Location: src/Tools/Common/GM/Features/Families/FamilyScanner.cs — QueryMappableParamNamesForFamily Contract: Scan ONE instance per family for parameter names. Parameter definitions are family-level. Why: All instances share identical definitions. Scanning N instances adds O(N * views) cost for zero new information.

4. Deep Scan Is Manual Only (GM)

Location: wiki/developer-guide/tools/common/gm-spec.md Section 15.1 Contract: Deep scan (opening family documents via EditFamily) is NEVER auto-triggered. User must click "Scan Family". Why: EditFamily takes seconds per family. Auto-triggering during usage build would stall the tool.

5. RevitCompat.IsUserMappable Filter Chain (Core)

Location: src/DBTools.Core/Revit/Utilities/RevitCompat.cs Contract: Parameters are filtered by: (1) not a built-in parameter, (2) not read-only, (3) UserModifiable property (when available via reflection). Why: Removing any filter exposes non-modifiable parameters across ALL tools. Users see parameters they cannot write to; commits fail at apply time.

6. Style Canonicalization is Simple (GM)

Location: src/Tools/Common/GM/Features/Usage/UsageIndexService.cs — RecordStyle, BuildStyleCanonicalizationMap Contract: Preserve raw style IDs and canonical projection IDs only (no broad alias fan-out). Why: Full alias fan-out (recording all equivalent IDs) multiplies dictionary insertions per geometry object without improving correctness; preserving raw + canonical IDs keeps verification/import compatibility without restoring broad expansion.

7. Style Maps Built Once Per Usage Build (GM)

Location: src/Tools/Common/GM/Features/Usage/UsageIndexService.cs — BuildStyleUsageAsync Contract: BuildStyleCanonicalizationMap and ResolveCanonicalHiddenLinesStyleId run ONCE and results are passed to chunk calls. Why: Rebuilding per 25-instance chunk means hundreds of FilteredElementCollector full scans for no reason.
